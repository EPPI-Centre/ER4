On this page you can set up tasks to find new related papers, which will appear here, whenever MAG is updated (every two weeks, usually).

Auto updates create a model of the items you will pick and apply it to new MAG references, so to indentify those new references, recently indexed in MAG, which appear to be similar to the items you currently have in this review.

There are two tables below: the first one shows each "auto update task" you have created (you will normally need only one).
The second table will automatically create a new entry (for each row in the first table) every time a new version of MAG is published. We call these "task runs", as they are "find new related papers" tasks that run automatically, when MAG receives an update.
These then show as "task results" (second table) and will indicate how many new references have been found. You will then be able to:

  • List the results without importing them
  • Import (and then screen normally) all the items found
  • Refine the results, by applying additional classifiers and/or picking more stringent thresholds (before importing and screening).

You can refine your results by also applying a "study" type model, or any one of your own "custom build models".
Beware of being "too clever", when applying machine learning models to reduce the number of new items: if your review currently "misses" some relevant studies, you could be perpetuating the same mistake at each update.
